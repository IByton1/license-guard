# Security Policy

The latest minor release receives security fixes. Do not disclose suspected false negatives,
path-handling vulnerabilities, or supply-chain issues in a public issue before maintainers have had
an opportunity to investigate.

Until a repository security contact is configured, keep a minimized private reproduction and open
a GitHub security advisory after the canonical repository is published. Never include proprietary
lockfiles, package sources, credentials, or registry tokens without explicit authorization.
