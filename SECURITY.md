# Security Policy

The latest minor release receives security fixes. Do not disclose suspected false negatives,
path-handling vulnerabilities, or supply-chain issues in a public issue before maintainers have had
an opportunity to investigate.

Use [GitHub private vulnerability reporting](https://github.com/IByton1/license-guard/security/advisories/new)
with a minimized reproduction. If private reporting is unavailable, open an issue requesting a
private contact without disclosing technical details. Never include proprietary lockfiles, package
sources, credentials, or registry tokens without explicit authorization.
